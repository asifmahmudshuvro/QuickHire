# QuickHire

QuickHire is a full-stack job board application built with Next.js (frontend) and Laravel (backend API), backed by MySQL.

## Tech Stack

- Frontend: Next.js (App Router), TypeScript, Tailwind CSS
- Backend: Laravel, Sanctum authentication
- Database: MySQL

## Core Features

- Public job listing with search and location/category filtering
- Job detail page with application form
- Candidate authentication (signup/login/logout)
- Admin authentication and protected admin dashboard
- Admin job management (create/delete)
- Admin application review section

## Project Structure

- `frontend/` - Next.js UI
- `backend/` - Laravel API

## Local Setup

### 1) Backend (Laravel)

From `backend/`:

1. Install dependencies:
   - `composer install`
2. Create `.env` and configure DB:
   - `DB_CONNECTION=mysql`
   - `DB_HOST=127.0.0.1`
   - `DB_PORT=3306`
   - `DB_DATABASE=quickhire`
   - `DB_USERNAME=root`
   - `DB_PASSWORD=`
3. Generate key:
   - `php artisan key:generate`
4. Run migrations + seed:
   - `php artisan migrate --seed`
5. Start server:
   - `php artisan serve`

Backend base URL: `http://127.0.0.1:8000/api`

### 2) Frontend (Next.js)

From `frontend/`:

1. Install dependencies:
   - `npm install`
2. Create `.env.local`:
   - `NEXT_PUBLIC_API_URL=http://127.0.0.1:8000/api`
3. Start dev server:
   - `npm run dev`

Frontend URL: `http://localhost:3000`

## Environment Variables

- Backend (`backend/.env`): standard Laravel variables + `DB_*` for MySQL
- Frontend (`frontend/.env.local`):
  - `NEXT_PUBLIC_API_URL` (required)

## User Types, IDs, and Credentials

- Admin user
  - Login ID (email): `admin@quickhire.test`
  - Password: `admin12345`
  - Role flag: `is_admin = true`
  - Numeric user ID (`users.id`) is auto-generated by MySQL

- Candidate user
  - Login ID: email used during signup
  - Password: password set during signup
  - Role flag: `is_admin = false`
  - Numeric user ID (`users.id`) is auto-generated by MySQL

Notes:
- Both user types are stored in a single `users` table.
- Role is determined by `is_admin`, not by a separate table or ID range.

## Main API Endpoints

Jobs:
- `GET /api/jobs`
- `GET /api/jobs/{id}`
- `POST /api/jobs` (admin only)
- `DELETE /api/jobs/{id}` (admin only)

Applications:
- `POST /api/applications`
- `GET /api/applications` (admin only)

Auth:
- `POST /api/auth/login`
- `GET /api/auth/me`
- `POST /api/auth/logout`
- `POST /api/auth/user/register`
- `POST /api/auth/user/login`
- `GET /api/auth/user/me`
- `POST /api/auth/user/logout`
